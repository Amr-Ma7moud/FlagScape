# FlagScape Project Stakeholders

## 1. Primary Stakeholders

### 1.1 Students (End Users)
**Role**: Primary users of the application

**Interests**:
- Easy-to-use, engaging platform for learning geography
- Fun and interactive games
- Accessible on various devices (desktop, mobile, tablet)
- Immediate feedback on quiz answers
- Visual and interactive learning experience

**Requirements**:
- Intuitive user interface
- Fast loading times
- Mobile-responsive design
- Clear instructions and feedback
- Variety of game modes to maintain interest

**Impact**: Direct beneficiaries of the educational value provided by the platform

---

### 1.2 Course Instructors (Academic Evaluators)
**Role**: Evaluate the project against course rubric

**Interests**:
- Demonstration of technical proficiency in web technologies
- Proper implementation of course requirements:
  - XHTML/HTML5 standards
  - Separation of concerns (HTML, CSS, JS)
  - jQuery usage
  - Client and server-side validation with regex
  - Float-based CSS layouts
  - DOM manipulation and event handling
  - AJAX/JSON data handling
  - Server-side programming (Python/Flask)
  - Database integration
- Code quality and organization
- Complete documentation

**Requirements**:
- All rubric items must be addressed (25 points total)
- Well-documented code
- UML diagrams showing system design
- Clear demonstration of technical concepts
- Proper validation (client and server-side)
- Evidence of unobtrusive JavaScript

**Impact**: Determine project grade and assess learning outcomes

## 7. Potential Stakeholder Conflicts

### Conflict 1: Feature Scope vs. Deadline
- **Stakeholders**: Development Team vs. Instructors
- **Issue**: Limited time to implement all desired features
- **Resolution**: Prioritize rubric requirements, implement core features first

### Conflict 2: User Experience vs. Code Simplicity
- **Stakeholders**: Students (UX) vs. Development Team (complexity)
- **Issue**: Advanced UX features may require complex implementation
- **Resolution**: Balance usability with maintainable code

### Conflict 3: Educational Depth vs. Entertainment
- **Stakeholders**: Geography Educators vs. Students
- **Issue**: Making content engaging while maintaining educational value
- **Resolution**: Gamification while ensuring accurate information
